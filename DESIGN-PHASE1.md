### Enums — `Models/Enums/PriorityLevel.cs` (fix #13)

`Priority` was a magic string ("HIGH" / "NORMAL" / "LOW") — replaced with a typed enum so the compiler catches typos and the frontend can exhaustively switch on it.

```csharp
namespace DeliveryTracker.API.Models.Enums;

public enum PriorityLevel
{
    Low,
    Normal,
    High
}
```

Stored as string in SQLite (same `HasConversion<string>()` pattern as `DeliveryStatus`).

### Entity — `Models/Entities/Delivery.cs`

| Field | Type | Notes |
|---|---|---|
| `Id` | `int` | Primary key, auto-increment |
| `DeliveryNumber` | `string` | e.g. `DLV-2841`, unique |
| `DestinationAddress` | `string` | Full delivery address |
| `RecipientName` | `string` | e.g. "BuildCo Inc." |
| `PackageWeight` | `decimal` | In lbs |
| `Priority` | `PriorityLevel` | Enum stored as string — `Low` / `Normal` / `High` (fix #13) |
| `Status` | `DeliveryStatus` | Stored as string in SQLite |
| `DriverId` | `string` | e.g. "D-07" |
| `DriverName` | `string` | e.g. "Marcus Thompson" |
| `OriginAddress` | `string` | "Trackr Warehouse, 4200 6th Ave S, Seattle" |
| `EstimatedMinutes` | `int` | ETA in minutes from now |
| `TotalRouteDistanceMiles` | `decimal` | Total route distance |
| `RouteWaypoints` | `string` | JSON array of waypoint objects — used by simulator (see Seed Data for format) |
| `CurrentWaypointIndex` | `int` | Which waypoint the simulator is currently at — updated each tick (**fix #1**) |
| `CurrentLatitude` | `double` | Last known latitude — updated each simulator tick (**fix #1**) |
| `CurrentLongitude` | `double` | Last known longitude — updated each simulator tick (**fix #1**) |
| `DispatchedAt` | `DateTime` | UTC timestamp when delivery was dispatched — renamed from `CreatedAt` for clarity (fix #15) |

> **Why `CurrentLat/Lon/WaypointIndex` on `Delivery` instead of always querying `Locations`?**
> The simulator runs every 3s for 12 deliveries — querying `MAX(Timestamp)` from `Locations` per delivery on every tick causes 12 extra queries. Storing current position on the entity itself means one `UPDATE` per tick, and `GET /api/deliveries` returns current position without joins.

```csharp
namespace DeliveryTracker.API.Models.Entities;

public class Delivery
{
    public int Id { get; set; }
    public string DeliveryNumber { get; set; } = string.Empty;
    public string DestinationAddress { get; set; } = string.Empty;
    public string RecipientName { get; set; } = string.Empty;
    public decimal PackageWeight { get; set; }
    public PriorityLevel Priority { get; set; } = PriorityLevel.Normal;  // fix #13
    public DeliveryStatus Status { get; set; } = DeliveryStatus.Pickup;
    public string DriverId { get; set; } = string.Empty;
    public string DriverName { get; set; } = string.Empty;
    public string OriginAddress { get; set; } = "Trackr Warehouse, 4200 6th Ave S, Seattle";
    public int EstimatedMinutes { get; set; }
    public decimal TotalRouteDistanceMiles { get; set; }
    public string RouteWaypoints { get; set; } = "[]";

    // Simulator state — updated every 3s tick (fix #1)
    public int CurrentWaypointIndex { get; set; } = 0;
    public double CurrentLatitude { get; set; }
    public double CurrentLongitude { get; set; }

    public DateTime DispatchedAt { get; set; } = DateTime.UtcNow;  // fix #15: renamed from CreatedAt

    public ICollection<Location> Locations { get; set; } = new List<Location>();
}
```

**`DeliveryDto.cs`** — flattened view returned by API
```csharp
public record DeliveryDto(
    int Id,
    string DeliveryNumber,
    string DestinationAddress,
    string RecipientName,
    decimal PackageWeight,
    string Priority,                // string form of PriorityLevel enum: "Low"/"Normal"/"High" (fix #13)
    string Status,                  // string form of DeliveryStatus enum
    string DriverId,
    string DriverName,
    int EstimatedMinutes,
    decimal TotalRouteDistanceMiles,
    double CurrentLatitude,         // from Delivery.CurrentLatitude (fix #1, #5)
    double CurrentLongitude,        // from Delivery.CurrentLongitude (fix #1, #5)
    int CurrentWaypointIndex,       // for progress bar calculation (fix #5)
    int TotalWaypoints,             // derived from RouteWaypoints array length (fix #5)
    DateTime DispatchedAt           // fix #15: renamed from CreatedAt
);
```

> `TotalWaypoints` is computed when mapping: `JsonSerializer.Deserialize<List<Waypoint>>(delivery.RouteWaypoints)?.Count ?? 0`
> `RouteProgress %` in the UI = `(CurrentWaypointIndex / TotalWaypoints) * 100`

**`LocationDto.cs`**
```csharp
public record LocationDto(
    int DeliveryId,
    string DeliveryNumber,  // included so SignalR broadcast is self-contained (fix #6)
    string Status,          // current delivery status at time of location record (fix #6)
    double Latitude,
    double Longitude,
    DateTime Timestamp,
    int WaypointIndex
);
```

> **Why `DeliveryNumber` + `Status` on `LocationDto`?** The SignalR hub broadcasts `LocationDto` to all dashboard clients on every 3s tick. Without these fields, the Angular map component would need a separate `GET /api/deliveries/{id}` call just to update the status badge — that is 12 extra HTTP calls every 3 seconds. Including them in the broadcast makes the payload self-contained.

## Authentication Flow

```
Angular LoginComponent
    │
    ├── POST /api/auth/login  { email, password }
    │
    ▼
AuthController
    │
    ├── AuthService.ValidateUser(email, password)
    │       └── BCrypt.Verify(password, user.PasswordHash)
    │
    ├── AuthService.GenerateToken(user)
    │       └── JwtSecurityToken
    │               issuer:   "trackr-api"
    │               audience: "trackr-client"
    │               claims:   email, role, displayName, sub (userId)
    │               expires:  DateTime.UtcNow + 8 hours
    │               signingKey: from appsettings.Development.json
    │
    └── Returns LoginResponseDto { token, email, displayName, role, expiresAt }

Angular AuthService
    └── Stores token in localStorage["trackr_token"]
    └── Decodes payload → sets currentUser signal
    └── Navigates to /dashboard
```

## CORS Configuration

```csharp
// Program.cs
builder.Services.AddCors(options =>
    options.AddPolicy("trackr", policy =>
        policy.WithOrigins("http://localhost:4200")
              .AllowAnyHeader()
              .AllowAnyMethod()
              .AllowCredentials()  // required for SignalR
    ));

// Must appear BEFORE app.UseAuthentication()
app.UseCors("trackr");
```

**Why `AllowCredentials()`?** SignalR sends the JWT in the WebSocket handshake. Without `AllowCredentials`, the browser blocks the connection. `AllowAnyOrigin` cannot be combined with `AllowCredentials` — explicit origin required.

## Program.cs Wiring

> **Fix #2:** This is the authoritative starter code for Phase 1A. It reads CORS origins from config (fix #3), includes the SignalR JWT query-string handler stub ready for Phase 2 (gap #2), and follows the correct middleware order.

```csharp
using System.Text;
using DeliveryTracker.API.Data;
using DeliveryTracker.API.Middleware;
using DeliveryTracker.API.Services;
using Microsoft.AspNetCore.Authentication.JwtBearer;
using Microsoft.EntityFrameworkCore;
using Microsoft.IdentityModel.Tokens;
using Microsoft.OpenApi.Models;

var builder = WebApplication.CreateBuilder(args);

// ── 1. Controllers + Swagger ────────────────────────────────────────────────
builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen(c =>
{
    c.SwaggerDoc("v1", new OpenApiInfo { Title = "Trackr API", Version = "v1" });
    c.AddSecurityDefinition("Bearer", new OpenApiSecurityScheme
    {
        Name = "Authorization",
        Type = SecuritySchemeType.Http,
        Scheme = "bearer",
        BearerFormat = "JWT",
        In = ParameterLocation.Header,
        Description = "Enter JWT token"
    });
    c.AddSecurityRequirement(new OpenApiSecurityRequirement
    {{
        new OpenApiSecurityScheme
        {
            Reference = new OpenApiReference { Type = ReferenceType.SecurityScheme, Id = "Bearer" }
        },
        Array.Empty<string>()
    }});
});

// ── 2. Database ──────────────────────────────────────────────────────────────
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlite(builder.Configuration.GetConnectionString("DefaultConnection")));

// ── 3. Authentication — JWT Bearer ──────────────────────────────────────────
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer           = true,
            ValidateAudience         = true,
            ValidateLifetime         = true,
            ValidateIssuerSigningKey = true,
            ValidIssuer              = builder.Configuration["Jwt:Issuer"],
            ValidAudience            = builder.Configuration["Jwt:Audience"],
            IssuerSigningKey         = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Secret"]!))
        };

        // SignalR JWT stub — reads token from query string for WebSocket connections
        // Activated in Phase 2 when LocationHub is added
        options.Events = new JwtBearerEvents
        {
            OnMessageReceived = ctx =>
            {
                var token = ctx.Request.Query["access_token"];
                if (!string.IsNullOrEmpty(token) &&
                    ctx.HttpContext.Request.Path.StartsWithSegments("/hubs"))
                    ctx.Token = token;
                return Task.CompletedTask;
            }
        };
    });

// ── 4. CORS — reads allowed origins from appsettings.json (fix #3) ──────────
var allowedOrigins = builder.Configuration
    .GetSection("Cors:AllowedOrigins")
    .Get<string[]>() ?? Array.Empty<string>();

builder.Services.AddCors(options =>
    options.AddPolicy("trackr", policy =>
        policy.WithOrigins(allowedOrigins)
              .AllowAnyHeader()
              .AllowAnyMethod()
              .AllowCredentials())); // required for SignalR

// ── 5. Application Services ──────────────────────────────────────────────────
builder.Services.AddScoped<IAuthService, AuthService>();

var app = builder.Build();

// ── 6. Seed database on startup ──────────────────────────────────────────────
using (var scope = app.Services.CreateScope())
    await SeedData.InitializeAsync(scope.ServiceProvider);

// ── 7. Middleware pipeline (ORDER MATTERS) ───────────────────────────────────
app.UseMiddleware<ExceptionHandlerMiddleware>();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseCors("trackr");        // BEFORE UseAuthentication
app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();

app.Run();
```

## Seed Data

**5 deliveries** seeded across the Bellevue, WA area (fix #8 — doc previously said 12 but seed implementation has 5; 5 is the correct number for Phase 1A). Each has:
- Unique `DeliveryNumber` (DLV-2837 through DLV-2841 — matching the dashboard mock)
- A mix of statuses: 2 `InTransit`, 1 `Nearby`, 1 `Pickup`, 1 `Delayed`
- A `RouteWaypoints` JSON array of **10 waypoints** along real Bellevue streets (fix #14 — 5 waypoints = 15s demo at 3s/tick, too short; 10 waypoints = 30s which is long enough to demo live movement)
- A driver from the pool: D-02 through D-11

### Waypoint Format — Objects throughout (fix #4)

All waypoints use the **object format** `{ "lat": ..., "lon": ... }` everywhere — in `SeedData.cs`, in `LocationSimulatorService`, and in any documentation. The array format `[lat, lon]` is banned to avoid index confusion.

A `Waypoint` record is defined for deserialization:
```csharp
// Models/Waypoint.cs
public record Waypoint(double Lat, double Lon);
```

Deserialized in the simulator with:
```csharp
var waypoints = JsonSerializer.Deserialize<List<Waypoint>>(delivery.RouteWaypoints)!;
```

## Angular App Structure

### `tsconfig.json` — key settings for Leaflet (fix #10)

> **Do NOT add `"types": ["leaflet"]`** — this restricts TypeScript to only the listed type packages and breaks `@angular/core`, `rxjs`, and every other `@types/*` package in the project. The correct fix is `skipLibCheck: true` only, which tells TypeScript to skip type-checking `.d.ts` files in `node_modules`.

```json
{
  "compilerOptions": {
    "skipLibCheck": true,
    "strict": true,
    "target": "ES2022",
    "module": "ES2022"
  }
}
```

Leaflet types are included by installing `@types/leaflet` as a dev dependency — no `types` array needed.

### `angular.json` — disable SSR, add Leaflet CSS
```json
{
  "styles": [
    "src/styles.scss",
    "node_modules/leaflet/dist/leaflet.css"
  ]
}
```

## AuthService Design

```typescript
// core/services/auth.service.ts
@Injectable({ providedIn: 'root' })
export class AuthService {
  private readonly TOKEN_KEY = 'trackr_token';

  // Angular signal — reactive, no BehaviorSubject needed
  currentUser = signal<UserModel | null>(this.loadUserFromToken());

  constructor(private http: HttpClient, private router: Router) {}

  login(email: string, password: string): Observable<void> {
    return this.http
      .post<LoginResponse>(`${environment.apiUrl}/api/auth/login`, { email, password })
      .pipe(
        tap(response => {
          localStorage.setItem(this.TOKEN_KEY, response.token);
          this.currentUser.set(this.decodeToken(response.token));
        }),
        map(() => void 0)
      );
  }

  logout(): void {
    localStorage.removeItem(this.TOKEN_KEY);
    this.currentUser.set(null);
    this.router.navigate(['/login']);
  }

  getToken(): string | null {
    return localStorage.getItem(this.TOKEN_KEY);
  }

  isAuthenticated(): boolean {
    const token = this.getToken();
    if (!token) return false;
    try {
      // JWT payload is base64url-encoded. The `exp` claim is Unix epoch seconds (not ms).
      // Multiply by 1000 to convert to ms before comparing with Date.now().
      // atob() does not handle base64url padding — tokens without '=' padding still work
      // because JSON.parse tolerates the extra characters after the closing '}'.
      // If the token is malformed (not 3 dot-separated segments), split('.')[1] is undefined
      // and atob() throws — caught below and returns false. (fix #11)
      const payload = JSON.parse(atob(token.split('.')[1]));
      return !!payload && (payload.exp * 1000) > Date.now();
    } catch {
      // Malformed token — treat as unauthenticated and clear storage
      localStorage.removeItem(this.TOKEN_KEY);
      return false;
    }
  }

  private decodeToken(token: string): UserModel | null {
    try {
      const payload = JSON.parse(atob(token.split('.')[1]));
      return {
        email: payload['email'] ?? payload['sub'],
        role: payload['role'],
        displayName: payload['displayName'],
        exp: payload['exp']   // kept so isAuthenticated() can read it via currentUser signal
      };
    } catch { return null; }
  }

  private loadUserFromToken(): UserModel | null {
    const token = this.getToken();
    return token ? this.decodeToken(token) : null;
  }
}
```

## JWT Interceptor Design

```typescript
// core/interceptors/jwt.interceptor.ts
export const jwtInterceptor: HttpInterceptorFn = (req, next) => {
  const auth = inject(AuthService);
  const token = auth.getToken();
  if (token) {
    req = req.clone({
      setHeaders: { Authorization: `Bearer ${token}` }
    });
  }
  return next(req).pipe(
    catchError(err => {
      if (err.status === 401) auth.logout();   // auto-logout on expired token
      return throwError(() => err);
    })
  );
};
```

Register in `app.config.ts`:
```typescript
provideHttpClient(withInterceptors([jwtInterceptor]))
```

## Key Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| JWT storage | `localStorage` | Portfolio tradeoff — simpler DX; `httpOnly` cookie in production |
| Password hashing | BCrypt | Industry standard; `BCrypt.Net-Next` NuGet package |
| ORM | EF Core + SQLite | Zero-config local DB; easy swap to SQL Server for Azure |
| Angular pattern | Standalone components | Angular 19 standard; no NgModules needed |
| State management | Angular Signals | Built-in reactive primitive; no NgRx overhead for this scale |
| Lazy loading | `loadComponent` | Reduces initial bundle; standard Angular 19 pattern |
| Guard style | Functional `CanActivateFn` | Angular 19 recommended over class-based guards |
| HTTP interceptor | Functional `HttpInterceptorFn` | Angular 17+ recommended pattern |
| `DeliveryStatus` | Enum stored as string | Human-readable in SQLite; no migration needed to add new status |
| Route waypoints | JSON string in `Delivery` | Avoids a separate `Waypoint` table; parsed by simulator at runtime |
